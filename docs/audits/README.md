# Audit Archive

This folder holds historical audit artifacts, generated inventories, and one-off analysis that informed shipped refactors.

These files are kept for traceability. They are not part of the runtime surface, package surface, or host integration surface.

## Contents

- `GLOSSARY_CANDIDATES.md`
  Locked vocabulary audit input used during the v9 cut.
- `DIRECTIVE_PREFIX_INVENTORY.md`
  Locked naming audit input used during the v9 cut.
- `engine-runtime-boundary-map.md`
  Engine/runtime boundary analysis from the deeper reorganization pass.
- `engine-runtime-state-audit.csv`
  File-level state and caller inventory generated by `scripts/audit-engine-runtime-state.ts`.
- `nested-path-audit.csv`
  Nested numbered-path inventory generated by `scripts/audit-nested-paths.ts`.
- `schema-example-coverage.csv`
  Schema/example coverage report generated by `scripts/audit-example-coverage.ts`.
- `ui-mutation-coverage-audit.md`
  UI mutation-surface audit for the bounded operator workbench direction.
- `audience-feature-inventory.md`
  Audience inventory used during audience and scope positioning work.
- `audience-scope-trim.md`
  Audience trim notes produced alongside the audience inventory.
- `contracts-audit.csv`
  Historical contract audit output retained for traceability.

## Rule

If a new audit is useful for a refactor or cut, put it here or have its generator write here directly. Do not drop temporary audit output into the repo root.
