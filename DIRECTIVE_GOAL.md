# Directive Goal

Keep the current host-project goal here.

Directive Kernel should read this file as the human-facing goal source of truth, then normalize it into the goal envelope described in `shared/contracts/goal-input.md`.

If this file is missing, hosts should fall back to per-request goal input and keep Discovery review-first or queue-only until a stable goal resolver exists.

## Goal ID

project-current-goal

## Goal Statement

Improve the host project's active product direction with one bounded useful capability or one bounded engine improvement.

## Why Now

Current delivery pressure or an explicit operator request makes this goal worth pursuing now.

## Adoption Target

runtime

## Constraints

- stay bounded
- keep review explicit

## Success Signal

One bounded result is materially clearer, more reusable, or more adoptable than before without weakening review discipline.
